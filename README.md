# truck-detection

ML half of a **WSU Transportation Research Group** feasibility study: detect trucks in single-capture
**PlanetScope SuperDove (~3 m)** satellite imagery over Washington freight corridors, via the
**moving-echo** signal — a colour-separated **blue → red → green** streak a moving vehicle leaves because
the sensor captures its bands a fraction of a second apart. The reference method is a Keypoint R-CNN
(Adamiak et al. 2025).

## Layout

- **`src/`** — the pipeline:
  - `export_coco.py` — labels + GeoTIFFs → COCO keypoints + 64×64 chips
  - `train_keypoint_rcnn.py` / `train_keypoint_rcnn_jitter.py` — fine-tune the detector (v0 / jitter-augmented v1)
  - `detect_scene.py` — sliding-window full-scene inference → detections in UTM
  - `infer_keypoints.py` — chip-level eval + montage · `inspect_scene.py`, `make_road_mask.py` — utilities
- **`backend/`** — Flask API (real dataset counts + live detection) for the web console
- **`frontend/`** — React/Vite ML console (Dataset / Training / Inference / Spec tabs)
- **`docs/`** — see below
- **`Annotations-RGB.gpkg`** — the keypoint labels (16 vehicles / 47 points), plus the QGIS project `.qgz`
- **`imagery/`, `data/coco/`, `weights/`, `outputs/`** — data + derived artifacts (gitignored; regenerated by the scripts)

## Quickstart

```bash
python3 src/export_coco.py                 # labels + GeoTIFFs -> data/coco/ (chips + COCO)
python3 src/train_keypoint_rcnn_jitter.py  # train (run on CPU; MPS gives NaN) -> weights/
python3 src/detect_scene.py Stanwood_08_20260504 --weights weights/keypoint_rcnn_echo_jitter.pt
```

Web console: `python3 backend/server.py` (port 8787) + `cd frontend && npm run dev` (port 5173).
